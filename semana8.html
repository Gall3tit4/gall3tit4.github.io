Vite + React y Evaluación del Primer Parcial

Exploramos el entorno de desarrollo ultrarrápido Vite para aplicaciones React y realizamos nuestra evaluación del primer parcial.

¿Qué es Vite?

Vite es una herramienta de construcción y desarrollo frontend de próxima generación. Fue creada por Evan You (autor de Vue.js) para reemplazar herramientas tradicionales como Webpack, ofreciendo mayor velocidad y simplicidad. Su principal ventaja es su sistema de arranque instantáneo (server start) y la actualización en caliente ultrarrápida gracias a la utilización de módulos ES nativos (ESM).

Ventajas de usar Vite

  • Inicio instantáneo sin empaquetado inicial.
  • Actualización en caliente (HMR) extremadamente rápida.
  • Soporte integrado para TypeScript, JSX, CSS Modules y más.
  • Configuración mínima por defecto.
  • Basado en ESM (módulos ES), compatible con navegadores modernos.

¿Cómo instalar y usar Vite?

Puedes crear un proyecto Vite con React ejecutando los siguientes comandos: 

npm create vite@latest my-vite-app -- --template react
cd my-vite-app
npm install
npm run dev

Esto creará una estructura moderna de carpetas optimizada para React, permitiendo iniciar el servidor de desarrollo en segundos.

Evaluación del Primer Parcial

Esta evaluación se realizó durante la semana 8 y consistió en el desarrollo de dos aplicaciones utilizando React y Vite. El enfoque principal fue la programación por pares, fomentando el trabajo colaborativo, la estructuración en componentes y el uso de archivos externos. A continuación, se detallan los enunciados propuestos:

1. Aplicación de Estudiantes

Desarrollar una aplicación en React y Vite con al menos tres componentes: ComponentGeneral, ListaEstudiantes y Estudiante. En ComponentGeneral, se debe leer los datos desde un archivo estudiantes.json que contenga información de 20 estudiantes con campos como: id, university, name, birthdate, address y barcode.

En el componente ListaEstudiantes, se deben mostrar todos los estudiantes organizados en dos columnas para pantallas medianas o mayores (md), y en una sola columna para pantallas pequeñas. Cada Estudiante debe mostrarse como una tarjeta (card) con su información. El campo barcode es un string con 10 dígitos separados por 2 guiones (por ejemplo: 23--455676--89).

Debe utilizarse CSS puro para el diseño visual.

2. Aplicación de Cronómetro

Desarrollar una aplicación con React y Vite que permita crear un cronómetro funcional. El cronómetro se inicializa con tres campos numéricos: horas, minutos y segundos. Al presionar el botón "Iniciar", los campos deben convertirse en etiquetas (labels), y comenzar la cuenta regresiva en tiempo real actualizando el DOM.

Al terminar la cuenta (llegar a 00:00:00), se debe reproducir un sonido continuo usando el archivo BeepExamen.mp3 en un bucle infinito (loop). También se debe utilizar CSS puro para todo el estilo visual de la aplicación.

Duración del examen: 60 minutos. Modalidad: Programación por pares.
